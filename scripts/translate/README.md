# 高质量巴菲特致股东信翻译工具

这个目录包含用于翻译巴菲特致股东信的高质量工具，特别注重翻译质量和一致性。

## 文件说明

### `translate_with_quality.py`
主翻译脚本，提供以下功能：

1. **批量翻译** - 支持批量处理多个年份
2. **质量检查** - 自动检查术语一致性、数字翻译等问题
3. **质量报告** - 生成详细的翻译质量报告
4. **后处理** - 自动修复常见翻译问题
5. **伯克希尔专用术语表** - 确保金融术语翻译一致

## 使用方法

### 基本使用
```bash
# 翻译单个年份
python translate_with_quality.py 2023

# 翻译多个年份
python translate_with_quality.py 2023 2022 2021

# 指定输入输出目录
python translate_with_quality.py 2023 --input-dir ../data/parsed --output-dir ../data/parsed
```

### 输出文件
- 翻译文件: `data/parsed/{year}/sections_zh.json`
- 质量报告: `quality_reports/quality_report_{year}.md`

## 质量保证特性

### 1. 术语一致性检查
脚本自动检查以下术语的翻译一致性：
- 公司名称（伯克希尔、苹果、可口可乐等）
- 人物名称（巴菲特、芒格、阿贝尔等）
- 金融术语（浮存金、经营收益、GAAP等）
- 投资组合公司

### 2. 数字和货币检查
- 自动检查美元金额翻译
- 检查百分比和数字格式
- 确保数字在翻译中正确保留

### 3. 质量评分系统
每个翻译段落都会获得一个质量分数（0-100），基于：
- 术语一致性
- 长度比例合理性
- 标点符号完整性

### 4. 翻译后处理
- 自动修复引号格式
- 优化标点符号
- 清理多余空格

## 与现有脚本的区别

| 特性 | 现有脚本 (`parsing/translate_sections.py`) | 本脚本 (`translate/translate_with_quality.py`) |
|------|--------------------------------------------|-----------------------------------------------|
| 主要目标 | 快速批量翻译 | 高质量翻译 |
| 质量检查 | 基本 | 全面（术语、数字、格式） |
| 质量报告 | 无 | 详细Markdown报告 |
| 术语表 | 无 | 伯克希尔专用术语表 |
| 后处理 | 无 | 自动修复常见问题 |
| 适用场景 | 快速生成初稿 | 最终发布质量 |

## 集成AI API

要使用实际的AI翻译，需要修改`TranslationEngine`类：

1. 设置API密钥和环境变量
2. 实现`translate_batch`方法调用AI API
3. 配置适当的提示词和参数

## 最佳实践

1. **先快速后精细**：
   ```bash
   # 第一步：快速生成初稿
   python ../parsing/translate_sections.py 2023
   
   # 第二步：质量检查和优化
   python translate_with_quality.py 2023
   ```

2. **人工校对**：
   - 阅读质量报告，重点关注低分段落
   - 检查术语一致性建议
   - 验证数字和金额翻译

3. **迭代改进**：
   - 根据质量报告改进翻译
   - 更新术语表
   - 优化后处理规则

## 扩展和定制

### 添加新术语
在`TranslationQualityChecker.BERKSHIRE_TERMS`中添加新的术语映射。

### 自定义质量检查
扩展`TranslationQualityChecker`类添加新的检查规则。

### 调整质量评分
修改`calculate_quality_score`方法调整评分权重。

## 故障排除

### 常见问题
1. **API调用失败**：检查网络连接和API密钥
2. **术语不一致**：更新术语表
3. **质量分数低**：检查具体问题并人工校对

### 调试模式
可以临时添加调试输出来了解翻译过程：
```python
# 在translate_batch方法中添加
print(f"翻译段落 {order}: {text_en[:50]}...")
```

## 贡献指南

1. 发现翻译问题 → 更新术语表
2. 发现质量检查漏洞 → 扩展检查规则
3. 优化后处理 → 改进修复逻辑
4. 性能优化 → 改进批量处理
